
Summary:
**Key Idea** Meta-Harness automates the optimization of an LLM's **"harness"**—the external code that controls how the model stores, retrieves, and processes information. While previous automated optimizers compress feedback into short summaries or scalar scores, Meta-Harness gives an AI coding agent **unrestricted filesystem access**. The agent uses standard terminal commands to read past source code, scores, and complete execution traces. This deep access allows the AI to diagnose exact points of failure and rewrite the structural code of the harness, ultimately outperforming complex, human-engineered systems.

**How-to (Implementation Steps)**

1. **Initialize:** Define your target task, fix the underlying LLM, provide a basic starting harness (like a simple prompt script), and set up an empty filesystem to store logs.
2. **Configure the Agent:** Assign a highly capable coding agent (like Claude Code) with a clear natural-language "skill" instruction outlining its role, allowed commands, and directory structure.
3. **Inspect History:** At the start of each loop, the agent explores the filesystem to read previous iterations' code and raw execution traces, forming hypotheses about why they failed.
4. **Propose & Validate:** The agent writes new harness code. Run this through a lightweight validation script on a tiny sample to catch syntax errors before committing to a full run.
5. **Evaluate & Log:** Test the proposed code on a dedicated search dataset. Automatically save the new source code, evaluation scores, and full execution traces into a new folder in the filesystem.
6. **Iterate:** Repeat this cycle for a fixed number of iterations (e.g., 20-40). The system will return a Pareto frontier of the best-performing harnesses discovered.


In short, the skill must be able to improve  itself. 
Thus, recording the workflow as a file to feedback to the main agent, the main agent revises the skill to make the next workflow better.
