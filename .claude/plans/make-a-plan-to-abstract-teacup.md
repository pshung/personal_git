# Hybrid Sim Workspace Skeleton

## Context

The hybrid simulator (vsim + QEMU TCG plugin) currently lives across two
unrelated trees on disk: `/local/nick/vsim` (the SystemC+Verilator side,
including `hybrid/qemu_plugin/`) and `/local/nick/qemu_v5` (the QEMU 9.2.0
source). To make the project clonable on a fresh host, both halves need to be
pulled together under one workspace with a single `build.sh` orchestrator.

The user has published two upstream repos:
- `https://gitea.andestech.com/nick/hybrid_vsim` (vsim side, plugin source lives inside it)
- `https://gitea.andestech.com/nick/hybrid_qemu` (QEMU fork)

Outcome: `git clone --recursive .../hybrid_sim && cd hybrid_sim && ./build.sh`
yields a working `qemu-system-riscv64`, `vsim`, and `libhybrid_handoff.so`.

## Directory layout

```
~/work/hybrid_sim/
  .git/
  .gitmodules
  hybrid_qemu/                  # submodule (QEMU sources)
    build/                      # out-of-tree QEMU build
      qemu-system-riscv64
  hybrid_vsim/                  # submodule (vsim sources)
    build/                      # out-of-tree vsim build
      vsim
    hybrid/qemu_plugin/
      libhybrid_handoff.so      # built by the plugin Makefile, consumed by vsim e2e
  build.sh
  README.md
```

## Build script contract

`build.sh` is a thin orchestrator. Subcommands:

| Command           | Action |
|-------------------|--------|
| `./build.sh`      | All three: qemu, vsim, plugin (incremental) |
| `./build.sh qemu` | Configure (if needed) + build QEMU |
| `./build.sh vsim` | Configure (if needed) + build vsim with hybrid mode ON |
| `./build.sh plugin` | Build `libhybrid_handoff.so` under `hybrid_vsim/hybrid/qemu_plugin/` |
| `./build.sh test` | `ctest -L hybrid-e2e` from vsim build dir |
| `./build.sh clean`| Remove both `build/` dirs and the plugin `.so` |

Build commands (resolved relative to repo root `$ROOT`):

```
QEMU:
  cd $ROOT/hybrid_qemu
  [ -f build/build.ninja ] || (mkdir -p build && cd build && \
    ../configure --target-list=riscv64-softmmu --enable-plugins --disable-werror)
  ninja -C build qemu-system-riscv64

vsim:
  cd $ROOT/hybrid_vsim
  [ -f build/CMakeCache.txt ] || cmake -S . -B build -G Ninja \
    -DVSIM_ENABLE_HYBRID=ON \
    -DVSIM_QEMU_ROOT=$ROOT/hybrid_qemu
  cmake --build build -j$(nproc)

Plugin:
  make -C $ROOT/hybrid_vsim/hybrid/qemu_plugin \
    QEMU_ROOT=$ROOT/hybrid_qemu
```

Order: qemu first (vsim's hybrid CMake checks `qemu-plugin.h`), then vsim,
then plugin (needs both QEMU headers and vsim's `hybrid/include/`).

`set -euo pipefail` and color-tagged step banners; no fancy build system.

## Setup steps (one-shot, in order)

1. `mkdir -p ~/work/hybrid_sim && cd ~/work/hybrid_sim`
2. `git init -b main`
3. `git submodule add https://gitea.andestech.com/nick/hybrid_qemu hybrid_qemu`
4. `git submodule add https://gitea.andestech.com/nick/hybrid_vsim hybrid_vsim`
5. `git submodule update --init --recursive` (pulls vsim's nested submodules: systemc-cci, verilator, etc.)
6. Write `build.sh` (executable).
7. Write `README.md` (5-line quickstart).
8. `git add .gitmodules build.sh README.md && git commit -m "structural: workspace skeleton with hybrid_qemu/hybrid_vsim submodules"`

## Verification

1. `git clone --recursive .../hybrid_sim /tmp/check && cd /tmp/check && ./build.sh`
   completes with exit 0 and produces:
   - `hybrid_qemu/build/qemu-system-riscv64`
   - `hybrid_vsim/build/vsim`
   - `hybrid_vsim/hybrid/qemu_plugin/libhybrid_handoff.so`
2. `./build.sh test` runs the round-trip suite (`ctest -L hybrid-e2e`) and passes
   the 13 hybrid e2e tests already wired in `HybridConfig.cmake`.
3. `./build.sh clean && ./build.sh` re-builds from scratch successfully.
4. Re-running `./build.sh` is a no-op (nothing reconfigured, ninja/make say "no work to do").
